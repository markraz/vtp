Area Tool

This tool lets you indicate a rectangular area of interest.  It's useful for indicating where you want to operate on other layers, such as elevation, imagery, and vegetation.

Menus:

  • Area Tool
    • Set to Extents
      • Resizes the area tool to surround all of the currently loaded layers.
    • Set to Layer Extents
      • Resizes the area tool to surround the currently selected layer.
    • Numeric Values
      • Allows you to examine and manually enter the range of the area tool.
    • Merge & Resample Elevation
      • Displays the Merge and Resample Elevation dialog (see below)
    • Merge & Resample Imagery
      • Displays the Merge and Resample Imagery dialog (see below)
    • Generate Vegetation
      • See Vegetation for an description of how to use this command.
    • Request Layer from WFS
    • Request Layer from WMS
      • These are experimental commands.  If you know how to use a WFS or WMS, give them a try!

Toolbar:

  • The Area Tool () can be used to set the desired area directly with the mouse.  Click on any edge of the area tool to move that edge, or click inside to move the whole area without resizing.

Merge and Resample Elevation Dialog

You can choose where the output will go: To a new layer (in memory), To a file, or To a tileset (folder of tiles).

This dialog displays:

  • The size of the Save Area, in either meters or degrees
  • An estimated Sample factor, which is a guess based on looking at the X spacing of the first elevation layer.
  • The projected Size at that density, which shows you the size of a grid that would result from sampling the area at the indicated sample factor.  Assuming that your input layers are of the same density (spacing), this indicates the approximate grid size of the indicated area.

Output to a new layer or to a file:

You can change the sample factor to see the projected size change.  Once you have a good idea of what output size you want, specify it in the "X size" and "Y size" fields.

Note that there is a helpful "Powers of 2 + 1" constraint you can use to limit the output to sizes which are supported by Terrain LOD, which is highly desirable at visualization runtime.

For effective visualization it is recommended that you pick a output size that is at least as large as your input data, but within the limits of your computer's RAM.

Output to a folder of tiles (Tileset):

This option has no limit on the size of the data.  Press Tile Options for the Tiling Options dialog.  Give the output a filename, and set the number of rows and columns of the tile grid along with the tile size (LOD0 Size) which gives the highest resolution of each tile.  The Information display helps you see whether your tiling is a good approximation of the input data your are resampling.  For smoother visualization, is it good to keep the tile grid size large and the LOD0 Size small (512 or less) so that there are shorter pauses as each tile is paged.  Alternately, if smoothness isn't a concern, you can decrease the tile grid size, and increase the LOD0 Size, which will result in a drastically smaller number of files on your disk.

See Enviro: Tilesets for a description of how tilesets are handled at runtime.  If you are using a tileset for your elevation, you must have a tileset of the same grid size for the imagery.
