Where to Put Your Data

Put your data in the following directories:

Data Location
elevation data (BT and PRJ files)
 also .ini for elevation tilesets		 Data/Elevation
geospecific images (JPG, TIF, BMP files)
 also .ini for imagery tilesets		 Data/GeoSpecific
road data (RMF files) Data/RoadData
built structure data (VTST files) Data/BuildingData
vegetation (species.xml and VF files) Data/PlantData
vegetation billboard images Data/PlantModels
locations (LOC and VTAP) Data/Locations
raw (abstract) GIS files Data/PointData
skydome textures (hemispherical) Data/Sky
terrain definitions (XML) Data/Terrains
whole earth images for Earth view Data/WholeEarth

There are a few other folders used internally by the VTP software, including Data/Culture for standard culture models and textures, Data/Fonts for fonts, Data/GeoTypical for geotypical textures such as ocean and roads.  You won't need to add to or modify these folders.

Don't forget that in order to use the continuous LOD rendering, elevation files must be square-power-of-2-plus-1 in size.  For example:

513 x 513
1025 x 1025
2049 x 2049
4097 x 4097
8193 x 8193

For textures, see the page on Using Your Own Imagery.

Creating Your Own Terrain Settings

If you want to make your terrain show up as a new terrain in the Enviro startup dialog, then you can create your own .ini files in the Terrain folder on the data path.  One easy way to do this is to copy one of the existing .ini files (for example, Generic.ini) and edit it, either as a text file or with Enviro, to contain the settings you want.  The first line in the file ("Name") is the name that will appear in Enviro.
