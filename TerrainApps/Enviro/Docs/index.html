VTP Enviro Documentation

Enviro is the VTP runtime environment program.  It provides an interactive, realtime 3D navigation of your virtual terrain.  There are two primary view modes: Earth and Terrain.  At startup, you chose which view you want to start with, and you can switch at any time.

Earth View

At startup, select "Earth View" and press OK.  You are now viewing the planet from space.

How to Navigate:

  • To spin the viewpoint around the earth, hold down the middle mouse button.
  • To zoom in and out, use the shift key and the middle mouse button to drag up and down.

Each known terrain area will be shown as a small red rectangle on the earth.  To switch to a terrain, just click within its rectangle with the left mouse button.

Commands:

  • Earth - Show Time of Day
    • shows the earth illuminated by sunlight based on real, current time according to your computer's clock
  • Earth - Add Point Data
    • allows you to display a set of point data (from a Shapefile) on the Earth as small yellow points.
    • Try the file Data/PointData/vtp-users.shp to see the locations of the first 2260 VTP recipients.

Terrain View

At startup, to launch with Terrain:

  • Use "Select" to choose from the known terrain.
  • Use "Edit Properties" to view or edit the properties of the currently chosen terrain (the Terrain Creation Parameters dialog)

Press "OK".  The terrain will now load, which may take up to a minute.  Messages in the status bar will inform you of the loading progress.

Making Your Own Terrain

  • Put your data in the following directories:
    Data Location
    elevation data (BT and PRJ files) Data/Elevation
    geospecific images (BMP or JPG files) Data/GeoSpecific
    road data (RMF files) Data/RoadData
    built structure data (VTST files) Data/BuildingData
  • Don't forget that in order to use the continuous LOD rendering, elevation files must be square-power-of-2-plus-1 in size.  For example:
    513 x 513
    1025 x 1025
    2049 x 2049
    4097 x 4097
    8193 x 8193
  • In the Startup Dialog, use the "Generic Terrain" setting, press "Edit Properties", and select your data in the Parameters dialog
  • If you want to make your terrain show up as a new terrain in the Enviro startup dialog, then you can create your own .ini files in the Terrain folder on the data path.  One easy way to do this is to copy one of the existing .ini files (for example, Generic.ini) and edit it, either as a text file or with Enviro, to contain the settings you want.  The first line in the file ("Name") is the name that will appear in Enviro.

Using Your Own Imagery (Textures)

  • Your image needs to match your elevation in both projection and extents.  One way to do this is to use VTBuilder to re-project and crop your elevation, as needed, to match your image.
  • Your imagery should be converted to either the BMP or JPG format.
  • Learn how large a texture your computer can display.  From the Enviro Start dialog, press "OpenGL Info..." and look at the number for "Maximum Texture Dimension"
  • If your terrain texture map is not larger than the maximum texture, you can use the "Single texture" option, otherwise you will need to use "4x4 Tiled texture".
  • Give your texture a filename ending in it's size, e.g. "newyork_1024.bmp"
  • Put your texture in in the directory Data/GeoSpecific
  • For a Single texture:
    • Your texture size must be a square power of 2, up to the limit of your computer (e.g. 512, 1024)
  • For a 4x4 Tiles texture:
    • You provide a single large square image file, and Enviro will break it up for you, at runtime, into a set of 16 tiles (4x4)
    • The size of each tile must be a power of 2, up to the limit of your computer (e.g. 512, 1024)
    • Due to pixel overlap to avoid texture seams, the dimensions may seem somewhat unusual - e.g. 2045 for a 4x4 at 512, 4093 for a 4x4 at 1024
    • The filename is made up of two parts: the prefix (e.g. "hawaii_") and the image size (e.g. "4093"). The dialog lets you specify the prefix, and it computes and appends the image size, based on the tile size you specify
    • So, when you're preparing your image files, simply name them ending in a number that is their size in pixels (e.g. "hawaii_4093.bmp")

Data in a Different Location

  • If you want to put your Data directory in another location, edit your Enviro.ini file and change the first line to contain the path to the data.  The path can be relative or absolute, and should leave on a trailing slash.  For example, the default path is:
        DataPath ../Data/
    You could change it to paths such as these, depending on your platform:
        DataPath ../../Data/
        DataPath C:\MyData\
        DataPath /home/me/mydata/
  • If you have data in different places, you can simply put more than one "DataPath" line into the Enviro.ini file, and all the data on all the paths will be available to Enviro.

Notes for Developers

  • If you want to fully customize the terrain, such as adding your own custom objects and behaviors to the scene
    1. create a sub-class of vtTerrain, using one of the existing subclasses (like TransitTerrain) as an example
    2. in Enviro.cpp, add a line to the function LoadTerrainDescriptions(), so that your .ini file will be constructed with your terrain class
