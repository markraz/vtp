Export Area Tool

The export area is a tool you can use to indicate a rectangular area of interest.  You can then use this area to indicate where you want to generate and export elevation or vegetation.

Menus:

  • Export Area
    • Set to Extents
      • Moves the black lines to surround all of the currently loaded terrains.
    • Numeric Values
      • Allows you to examine and manually enter the range of the export area.
    • Merge & Export Elevation
      • Displays the Merge and Resample dialog (see below) and then asks for a filename to export elevation to.
    • Generate & Export Vegetation
      • see Vegetation for an description of how to use this command

Merge and Resample Dialog

This dialog displays:

  • The size of the Save Area, in either meters or degrees
  • An estimated Sample factor, which is a guess based on looking at the X spacing of the first elevation layer.
  • The projected Size at that density, which is the size of a grid that would result from sampling the area at the indicated sample factor.  Assuming that your input layers are of the same density (spacing), this indicates the approximate grid size of the indicated area.

You can change the sample factor to see the projected size change.  Once you have a good idea of what output size you want, specify it in the "X size" and "Y size" fields.

Note that there is a helpful "Powers of 2 + 1" constraint you can use to limit the output to sizes which are supported by Terrain LOD, which is highly desirable at visualization runtime.

For effective visualization, it is recommended that you pick a output size that is at least as large as your input data, but within the limits of your computer's RAM.
